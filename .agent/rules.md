# 📋 Antigravity 작업 원칙 (Standard Operating Procedures)

이 저장소에서 수행되는 모든 개발 및 관리 작업은 다음의 원칙을 **절대적으로 준수**한다. AI 어시스턴트는 매 작업 시작 전 이 파일을 로드하여 내용을 숙지해야 한다.

---

## 🚀 1. 작업 시작 필수 프로세스 (Mandatory Workflow)

모든 작업은 반드시 다음 단계를 순차적으로 수행한다.

1.  **규칙 로드**: `.agent/rules.md` 파일을 읽고 최신 원칙을 확인한다.
2.  **저장소 동기화**: 하루의 첫 작업 시 `git pull`을 통해 최신 코드를 확보한다.
3.  **의도 파악**: 사용자의 요청을 분석하되, **임의의 UI/UX 변경**이 포함되지 않도록 주의한다.
4.  **작업 수행**: 아래 정의된 세부 개발 원칙에 따라 코드를 수정한다.
5.  **검증 및 배포**: 버전 업데이트 규칙 준수 확인 후 커밋 및 푸시한다.

---

## 🛠 2. 개발 및 UI/UX 원칙

### 🧱 스켈레톤(Skeleton) 상태 유지
- 데이터 로딩 중이거나 결과가 없을 때, 임의로 "소식이 없습니다"와 같은 안내 메시지를 추가하지 않는다.
- 사용자의 별도 요청이 없는 한, 기존의 **스켈레톤 로딩 UI를 유지**하는 것을 원칙으로 한다.

### 🚫 임의 변경 금지
- 기능 개선이나 가독성 향상이라는 명목으로 기획서(요청사항)에 없는 UI 요소, 문구, 알림창을 마음대로 추가하거나 변경하지 않는다.

---

## 🔢 3. 버전 관리 및 배포 규칙 (Versioning)

배포 시 `src/components/SiteFooter.vue`의 버전 번호를 반드시 업데이트한다.

-   **버전 형식**: `Major.Minor` (예: `21.5`)
    -   **중요**: 반드시 **소수점 한 자리**까지만 사용한다.
    -   ❌ 잘못된 예: `21.5.1`, `21.5.0.1`
    -   ✅ 올바른 예: `21.6`, `22.0`
-   **업데이트 기준**:
    -   **Minor (소수점 첫째 자리)**: 버그 수정, 스타일 수정, 기능 추가 등 일반적인 모든 작업.
    -   **Major (정수 부분)**: 대규모 구조적 변경. **(반드시 사용자에게 사전 확인 필수)**
-   **배포 절차**: 버전 업데이트와 코드 수정을 한꺼번에 커밋하여 푸시한다.

---

## 📝 4. 커밋 메시지 규칙

작업의 성격에 따라 다음 접두어를 사용하며, 버전 정보를 포함한다.

-   `fix:`: 버그 수정
-   `feat:`: 기능 추가
-   `style:`: 코드 의미에 영향을 주지 않는 변경 (CSS, 포맷팅 등)
-   `docs:`: 문서 수정 (`rules.md` 등)
-   `chore:`: 빌드 업무, 패키지 매니저 설정 등

**예시**: `fix: blog loading issues and bump version to 21.6`

---

> [!IMPORTANT]
> **AI 어시스턴트 주의사항**: 사용자의 요청 사항 중 과거에 정의된 규칙(예: 스켈레톤 유지)이 있다면 이를 최우선으로 적용한다. 규칙을 어길 시 작업 생산성이 저하되므로 반드시 준수한다.
