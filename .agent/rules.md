# 작업 원칙 및 배포 가이드라인

본 저장소의 모든 작업 시 다음의 원칙을 반드시 준수한다.

## 1. 일일 첫 작업 전 동기화 (Daily First-work Sync)
- **하루 기준 처음** 공유 받은 작업이나 프롬프트 입력을 수행할 때, 원격 저장소와의 동기화를 확인한다. (매 프롬프트마다 수행할 필요는 없음)
- `git pull`을 통해 원격 저장소의 최신 변경 사항을 로컬로 가져온다.
- `/시작` 워크플로우를 활용하여 일자별 작업 시작 시 동기화를 처리한다.

## 2. 배포 시 버전 업데이트 (Deployment Versioning)
- 코드를 배포(Push)할 때마다 사이트의 버전 번호를 업데이트한다.
- **대상 파일**: `src/components/SiteFooter.vue` 내의 `Version` 정보.
- **업데이트 규칙**: 
  - 단순 버그 및 마이너 스타일 수정: `Patch` 버전 (예: 14.4 -> 14.4.1 또는 소수점 셋째 자리 활용)
  - 기능 추가 및 일반적인 변경: `Minor` 버전 (예: 14.4 -> 14.5)
  - **대규모 구조적 변경 및 메이저 업데이트**: (예: 14.x -> 15.0) **(이 경우에만 반드시 사용자에게 확인 후 업데이트 할 것)**
- **절차**: 코드 수정 완료 후 `SiteFooter.vue`의 버전을 규칙에 따라 조정한 뒤 한꺼번에 커밋하여 푸시한다.

## 3. 커밋 메시지 규칙
- 작업 내용에 따라 적절한 접두어를 사용한다 (예: `feat:`, `fix:`, `style:`, `chore:`).
- 버전 업데이트가 포함된 경우 커밋 메시지에 명시한다 (예: `fix: header jittering and bump version to 14.2`).
